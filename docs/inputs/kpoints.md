
## KPOINTS

The [KPOINTS](https://vasp.at/wiki/KPOINTS) file was formerly required by VASP but is now optional.
If you are using automatic grid generation, then you should try to prioritize the `KSPACING` and `KGAMMA` tags in the INCAR.
These are typically easier to control and create a symmetry-adapted grid, and will generally lead to fewer errors in generating a symmetry-appropriate basis set.

However, if you are doing a line-mode bandstructure calculation, or if you need to specify a shift in the $\bm{k}$-point grid, then you need to use the KPOINTS file.

### Format

The KPOINTS file can take on a wide variety of formats, you should generally allow a tool like `pymatgen` to set up and parse this file for you to avoid mistakes.

For example, performing either a meta-GGA or hybrid calculation of the band structure in one step requires "zero-weighted" $\bm{k}$-points to appropriately sample the high-symmetry path.

The general format of this file is:
1. Comment line
2. Number of $\bm{k}$-points specified explicitly
3. That the coordinates of the $\bm{k}$-points are specified in units of the reciprocal lattice vectors (`Reciprocal`, **not `Direct`**) or in cartesian coordinates (`Cartesian`)
4. The $x$, $y$, $z$ components of the $\bm{k}$-points, and optionally, their weight. You can see that `pymatgen` has labelled the high-symmetry $\bm{k}$-points. These labels are ignored by VASP.
